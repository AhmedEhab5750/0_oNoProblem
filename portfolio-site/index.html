> Race Condition Allows Users to Exceed Project Quota on All Plans_

A race condition in the project creation process allows users to bypass all plans quota limits by sending multiple parallel requests. Despite the enforced limit of 2 projects for free plans, an attacker can create unlimited projects across Development, Staging, and Production environments.

> Quota Bypass via Project Transfer Between Workspaces_

This vulnerability allows users to bypass the intended project quota limits for the Development plan (free), potentially affecting the application's revenue model and resource management. A malicious user could exploit this vulnerability to create an unlimited number of projects in a single workspace without upgrading to a paid plan, leading to resource abuse and loss of revenue.

> Race Condition Enables Bypass of Login and MFA OTP Rate Limiting Verification._

A race condition & response manipulation vulnerabilities have been identified in the authentication workflow, allowing attackers to bypass rate limiting on the login function and manipulate responses to circumvent Multi-Factor Authentication (MFA) via email-based One-Time Passwords (OTP). By sending multiple parallel requests using a single packet attack, an attacker can bypass rate limit and get user password and gain unauthorized access to user account without being subjected to rate limiting or account lockout mechanisms.
